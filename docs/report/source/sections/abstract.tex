\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}

\begin{sloppypar}
Sprout is an adaptive learning platform with a real-time multimodal tutor (voice, text, webcam, screen sharing). This report documents the production AI subsystem: a JWT-validated \emph{Gemini Proxy} on Cloud Run that connects browsers to Google Gemini Live, forwards bidirectional media, and records session telemetry. Replacing a LiveKit/WebRTC pipeline removes server-side audio hops and targets p95 round-trip latency below 500\,ms.

Firestore persists transcripts, completion state, and usage in the collections \code{users}, \code{courses}, \code{lessons}, \code{conversations}, \code{messages}, and \code{gemini\_sessions}, with planned per-course \code{course\_profiles} for multi-intent priming and parent dashboards.

We formalise six algorithms---authenticated session establishment, tokenised routing, knowledge-map generation via Cloud Tasks, dual-channel lesson completion, feedback summarisation, and anonymous progress promotion---and ship a TypeScript SDK with observability (Cloud Logging, Mixpanel, AgentOps). Evaluation is observability-first: improvements over the decommissioned LiveKit baseline rest on offline metrics and pilot estimates, not a finished production A/B test; Chapter~\ref{ch:evaluation} specifies a phased online protocol. For learners with multiple interests, we propose \code{course\_profiles} and \textsc{BuildSessionContext} priming in the backend only (Section~\ref{sec:multi-intent}). Future work adds function calling and coordinator--specialist multi-agent orchestration on the same gateway.
\end{sloppypar}

\noindent\textbf{Keywords:} multimodal LLM; Gemini Live API; WebSocket gateway; JWT; conversational tutor; adaptive learning; Firestore; multi-agent systems.
